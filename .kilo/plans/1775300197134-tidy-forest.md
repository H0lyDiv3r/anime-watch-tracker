# Plan: Integrate Supabase with Kotlin SDK in Android App

## Overview
Your codebase is an Android Jetpack Compose app for an anime list with basic auth screens. We'll integrate Supabase as the backend using the official Supabase Kotlin SDK (supabase-kt), which uses Ktor internally. This provides auth, database, and real-time capabilities with a clean API.

## Assumptions
- You're using the official Supabase Kotlin SDK for all backend interactions.
- The app will handle user authentication and anime list CRUD operations.
- Anime table: anime_list with columns id (UUID), title (String), description (String), user_id (UUID).
- No existing network layer; we'll build from scratch using the SDK.

## Prerequisites
- Supabase account and project (create at supabase.com).
- Project URL and anon/public API key from Supabase dashboard.

## Step-by-Step Integration Process

### 1. Add Dependencies
Update `app/build.gradle.kts`:
- Add Supabase Kotlin SDK: `implementation(platform("io.github.jan-tennert.supabase:bom:3.5.0"))`, `implementation("io.github.jan-tennert.supabase:postgrest-kt")`, `implementation("io.github.jan-tennert.supabase:auth-kt")`, `implementation("io.github.jan-tennert.supabase:realtime-kt")`.
- Add Coroutines for async: `implementation("org.jetbrains.kotlinx:kotlinx-coroutines-android:1.7.3")` (if not present)
- Note: The SDK uses Ktor internally.

### 2. Initialize Supabase Client
- Create a Supabase client instance with your project URL and anon key.
- Configure it in an Application class or dependency injection setup.

### 3. Define Data Models
- Create Kotlin data classes for anime items (id: String, title: String, description: String, user_id: String) that match your Supabase table schema.

### 4. Implement Repository Layer
- Create repository classes using the Supabase client for auth and database operations.
- Use supabase.auth for signIn/signUp.
- Use supabase.postgrest for CRUD on anime_list table.
- Handle exceptions and responses from the SDK.

### 5. Integrate Auth in UI
- In LoginScreen/SignupScreen composables, call repository auth methods on button clicks.
- The SDK handles token storage and refresh automatically.

### 6. Implement Anime List CRUD
- In main list screen, fetch anime list using supabase.postgrest.select().
- Add functionality to insert, update, delete anime items.
- Use StateFlow or MutableState for reactive UI updates.

### 7. Add Real-time (Optional)
- Use supabase.realtime to subscribe to changes in the anime_list table for live updates.

### 9. Security Considerations
- Never expose service_role key in client.
- Use anon key for public operations.
- Implement RLS (Row Level Security) in Supabase for data access control.

### 10. Testing and Debugging
- Test auth flow and CRUD operations.
- Use Supabase dashboard to verify data.
- Handle network errors gracefully in UI.

## Potential Challenges
- Ensure table schemas match data classes.
- Handle auth state changes and session management.
- Rate limits and error handling from SDK exceptions.
- Build errors: Ensure repositories include mavenCentral() in build.gradle.kts if dependencies can't be resolved.
